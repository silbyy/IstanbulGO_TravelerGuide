**ISTANBULGO - A DATA STRUCTURES PROJECT**

  We developed this project as a term project in our **Data Structures and Management** course so our main focus was demonstrating how        different data structures work together to solve real world problems

**1. Binary Search Tree (BST) - Area Management**

  We used a Generic BST to store and manage district (Area) data.
  We used this data structure to ensure efficient searching, insertion, and _alphabetical sorting_ of Istanbul's districts.
  Complexity: Provides an average of _O(logn) time complexity_ for search operations.

**2. Graph (Adjacency List) - The Transportation Network**

  Istanbul's transport system is modeled using a Weighted Graph.
  Nodes: Represent districts (Taksim, Kadikoy, etc.).
  Edges: Represent transport routes with weights representing travel time.
  Algorithm: We implemented _Dijkstra’s Algorithm_ to find the shortest path between any two points in the city.

**3. JSON & Linked Lists - Data Persistence**

  JSON: All location data (historical sites, cafes, etc.) is stored in a structured data.json file.
  Linked Lists: Used within the BST nodes to hold a dynamic list of locations for each specific district.

For Visual Design we used a custom minimalist edition UI

  Designed to be eye pleasing with the colors of #40262A and #7B4A52

**ISTANBULGO - VERİ YAPILARI PROJESİ**

Bu projeyi **Data Structures and Management** dersimiz için bir dönem projesi olarak geliştirdik; bu nedenle temel odak noktamız, farklı veri yapılarının gerçek dünya problemlerini çözmek için nasıl bir arada çalıştığını göstermekti.

**1. Binary Search Tree (BST) - Area Management**

Bölge (Area) verilerini saklamak ve yönetmek için Generic (Genel) bir BST kullandık.
Bu veri yapısını, İstanbul'un semtlerinin verimli bir şekilde aranmasını, eklenmesini ve alfabetik olarak sıralanmasını sağlamak için kullandık.
Karmaşıklık: Arama işlemleri için ortalama O(logn) zaman karmaşıklığı ile milisaniyeler içerisinde sonuca ulaşılmasını sağlamış olduk.

**2. Graph (Adjacency List) - The Transportation Network**

İstanbul'un ulaşım sistemini Ağırlıklı Weighted Graph kullanılarak modelledik.
Düğümler (Nodes): İlçeleri temsil eder (Taksim, Kadıköy, Beşiktaş vb.).
Kenarlar (Edges): Ağırlıkları seyahat süresini temsil eden ulaşım rotalarını temsil eder.
Algoritma: Şehirdeki herhangi iki nokta arasındaki en kısa yolu bulmak için Dijkstra Algoritmasını uyguladık.

**3. JSON & Linked Lists - Data Persistence**

JSON: Tüm konum verilerini (tarihi yerler, kafeler vb.) yapılandırılmış bir data.json dosyasında sakladık.
Bağlı Listeler: Her bir semte ait dinamik konum listelerini BST düğümleri içinde tutmak için kullandık.

Görsel Tasarım için özel bir minimalist sürüm kullanıcı arayüzü kullandık.

#40262A ve #7B4A52 renkleri ile göz yormayacak şekilde tasarlamak için çaba gösterdik.
